# Move Library — Configuration and Adaptive Defense

## Core idea
These moves let the player author likely futures through incomplete knowledge.

## Moves

### Reposition a turret for a narrower but stronger lane
**Why:** commit to a threat model.  
**Accomplishes:** better kill zone efficiency.  
**Cost / risk:** weaker general coverage.

### Keep a defense in reserve
**Why:** flexibility is valuable under uncertainty.  
**Accomplishes:** supports reactive adaptation.  
**Cost / risk:** lower passive strength now.

### Build for deterrence instead of damage
**Why:** buying time may matter more than kills.  
**Accomplishes:** reshapes pacing and routing.  
**Cost / risk:** threats may survive longer.

### Build for cleanup instead of first contact
**Why:** assume something will breach.  
**Accomplishes:** supports recovery from imperfection.  
**Cost / risk:** weaker initial stop power.

### Bias defense toward infrastructure instead of the player body
**Why:** sector function outranks comfort.  
**Accomplishes:** protects long-term viability.  
**Cost / risk:** personal danger rises.

### Bias defense toward the operator workspace
**Why:** if the player falls behind, everything worsens.  
**Accomplishes:** improves survivability and continuity of action.  
**Cost / risk:** outer line may collapse sooner.

### Prepare one ugly fallback plan
**Why:** brittle perfection is dangerous.  
**Accomplishes:** creates recoverability.  
**Cost / risk:** some resources are tied up for contingency.

### Intentionally abandon an outer node
**Why:** preserve the core.  
**Accomplishes:** concentrates defenses.  
**Cost / risk:** map control and confidence shrink.

### Set traps where you expect the threat to go
**Why:** defense is prediction.  
**Accomplishes:** rewards understanding.  
**Cost / risk:** low value if prediction is wrong.

### Set traps where you never want to travel
**Why:** reduce your own labor burden.  
**Accomplishes:** route denial and self-protection.  
**Cost / risk:** later salvage or intervention becomes harder.

### Reconfigure monitors to support a new theory
**Why:** information layout follows interpretation.  
**Accomplishes:** better support for current plan.  
**Cost / risk:** confirmation bias increases.

### Prepare for likely failure instead of likely success
**Why:** the game is about surviving bad nights.  
**Accomplishes:** stronger resilience.  
**Cost / risk:** lower maximum clean-win efficiency.

### Trust automation to do one job
**Why:** attention is scarce.  
**Accomplishes:** frees cognitive bandwidth.  
**Cost / risk:** failure feels personal if trust was misplaced.
