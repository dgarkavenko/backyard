# Performative Move Framework

## Working definition
A performative move is not only a direct player action.
It can also be:
- a deliberate interpretation
- a committed suspicion
- an act of postponement
- a feeling that changes later play
- a choice to trust, wait, sacrifice, or endure

## Criteria for a strong move
A move should:
- have a distinct and legible consequence
- change the state of the game or the player's understanding
- differ meaningfully from other moves
- be predictable enough to support learning
- remain uncertain enough to support anticipation

## Good move design questions
- Why would the player make this move?
- What does it accomplish?
- What cost does it carry?
- What kind of future does it create?
- What belief about the game does it reinforce?

## For this game, strong moves often trade
- time for certainty
- parts for future stability
- outer coverage for core reliability
- present calm for future fragility
- immediate response for informed response

## Internal performative moves for this game
Examples:
- wondering if a quiet sector is actually the real threat
- feeling guilty for sacrificing an outer node
- trusting automation instead of intervening
- deciding that a system can limp until morning
- reinterpreting a malfunction as part of a larger pattern
