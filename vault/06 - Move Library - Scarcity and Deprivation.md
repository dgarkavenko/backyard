# Move Library — Scarcity and Deprivation

## Core idea
These moves create the feeling that every action authors a future weakness.

## Moves

### Postpone an upgrade because repair comes first
**Why:** preserve function over growth.  
**Accomplishes:** keeps the line alive.  
**Cost / risk:** delays future strength.

### Spend rare parts on infrastructure instead of firepower
**Why:** tomorrow matters more than tonight.  
**Accomplishes:** strengthens resilience.  
**Cost / risk:** weaker short-term defense.

### Spend the last parts on immediate protection
**Why:** fear of the next breach.  
**Accomplishes:** stronger tonight.  
**Cost / risk:** crippled recovery if things still go badly.

### Hoard a resource for emergency use
**Why:** optionality has value.  
**Accomplishes:** preserves flexibility.  
**Cost / risk:** lower present effectiveness.

### Use an imperfect part because it is all you have
**Why:** improvisation under scarcity.  
**Accomplishes:** restores partial function.  
**Cost / risk:** unstable or inefficient outcome.

### Recover damaged components from ruined defenses
**Why:** waste is unacceptable.  
**Accomplishes:** partial material recovery.  
**Cost / risk:** exposure and time cost.

### Accept inefficiency to stay ahead of the clock
**Why:** speed is more important than elegance.  
**Accomplishes:** rapid setup.  
**Cost / risk:** long-term repair debt.

### Squeeze in one more task
**Why:** the outpost never feels ready.  
**Accomplishes:** marginal improvement.  
**Cost / risk:** overlooked warning signs or poorer choices.

### Give up on perfect preparation
**Why:** time itself is the pressure.  
**Accomplishes:** locks in a workable plan.  
**Cost / risk:** unresolved vulnerabilities remain.

### Choose which future problem to become
**Why:** not everything can be secured.  
**Accomplishes:** focuses resources where they matter most.  
**Cost / risk:** one chosen weakness is now authored intentionally.
